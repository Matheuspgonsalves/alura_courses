## English

This repository contains my Alura study materials. It works as a continuous learning space where I organize courses, career tracks, exercises, challenges, and projects built during my studies.

Its main purpose is to keep a practical record of my technical growth, covering both parallel learning paths and subjects directly connected to my professional work.

### How Alura is structured

On Alura, I can study in two main ways:

- by following a `career track`
- by taking a `standalone course`

When I choose a career track, the learning path usually follows this hierarchy:

- `career`
- `level`
- `course`
- `module`

Inside each module, there may be:

- `projects`
- `exercises`
- `challenges`

Not every module will necessarily include a project. A module may contain one project, multiple projects, or no project at all, depending on the course design or on my decision to focus only on the parts that match my current learning needs.

### How this repository will be organized

This repository follows a flexible structure inspired by how the platform itself is organized:

```text
alura_courses/
├── career_<topic>/
│   ├── level_<number>_<name>/
│   │   ├── course_<name>/
│   │   │   ├── module_<number>_<name>/
│   │   │   │   ├── project_<name>/
│   │   │   │   ├── exercises/
│   │   │   │   └── challenges/
│   │   │   └── README.md
│   │   └── challenge/
│   └── README.md
└── course_<name>/
```

### Organization principles

- `career tracks` and `standalone courses` may coexist in the same repository
- each folder can represent an active learning path
- projects are stored when they add practical value to the study process
- exercises and challenges may be separated by module or by course
- the structure may evolve over time as my study process changes

### Current status

At the moment, the repository already includes an initial structure focused on the Java Back-End career path, with materials organized by level and hands-on practice.

## Português

Este repositório reúne meus estudos na Alura. Ele funciona como uma base contínua de atualização profissional, onde organizo cursos, carreiras, exercícios, desafios e projetos que desenvolvo ao longo do aprendizado.

O objetivo é manter um histórico prático da minha evolução técnica, tanto em assuntos que complemento em paralelo quanto em temas diretamente ligados à minha atuação profissional.

### Como a Alura se organiza

Na Alura, posso estudar de duas formas:

- escolhendo uma `carreira`
- escolhendo um `curso isolado`

Quando a escolha é uma carreira, a estrutura normalmente segue esta hierarquia:

- `carreira`
- `nível`
- `curso`
- `módulo`

Dentro de cada módulo, podem existir:

- `projetos`
- `exercícios`
- `desafios`

Nem todo módulo necessariamente terá projeto. Em alguns casos pode haver um projeto, vários projetos ou nenhum, seja pela proposta do curso ou pela minha escolha de focar apenas no conteúdo que fizer sentido para meu momento de estudo.

### Como este repositório será organizado

Este repositório seguirá uma organização flexível, mas baseada na estrutura da plataforma:

```text
alura_courses/
├── carreira_<tema>/
│   ├── nivel_<numero>_<nome>/
│   │   ├── curso_<nome>/
│   │   │   ├── modulo_<numero>_<nome>/
│   │   │   │   ├── projeto_<nome>/
│   │   │   │   ├── exercicios/
│   │   │   │   └── desafios/
│   │   │   └── README.md
│   │   └── challenge/
│   └── README.md
└── curso_<nome>/
```

### Critérios de organização

- `carreiras` e `cursos isolados` podem coexistir no mesmo repositório
- cada pasta pode representar uma trilha real de estudo em andamento
- projetos serão salvos quando fizer sentido para o curso e para meu objetivo de prática
- exercícios e desafios também podem aparecer separados por módulo ou curso
- a estrutura pode evoluir com o tempo para refletir melhor meu ritmo de estudos

### Estrutura atual

Hoje o repositório já contém uma base inicial voltada para a carreira de Back-End Java, com exemplos organizados por nível e projetos práticos.
